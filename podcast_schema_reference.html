6. podcasts  (episodes)

Collection

Individual podcast episodes uploaded by Hosts. Maps to the Podcasts table in your ERD, extended with all missing required fields. Default approval_status is PENDING on upload.

Business rule: Episodes can only be deleted by the Host when approval_status is PENDING or REJECTED. APPROVED episodes cannot be deleted. Enforce this in the Express route handler.

9. episode_likes NEW — Missing from ERD

Collection

Normalised likes table — one row per user + episode pair. Prevents double-likes and supports unlike. When a row is inserted, also increment podcasts.likes_count. When deleted, decrement it.

Add a unique compound index on { user_id: 1, podcast_id: 1 } so the database enforces no duplicate likes even under concurrent requests.

AWS S3 — Key Naming Convention

Store ONLY the S3 object key in MongoDB (not the full URL). In Express, call getSignedUrlPromise() to generate a temporary pre-signed URL before returning the response to the client.

Asset TypeS3 Key PatternBucket ACLPre-signed TTL
Audio fileaudio/{userId}/{episodeId}.mp3Private15–60 minutes
Episode thumbnailcovers/episodes/{episodeId}/thumb.jpgPublic-readNo expiry (CDN)
Channel cover artcovers/channels/{channelId}/cover.jpgPublic-readNo expiry (CDN)
User avataravatars/{userId}/avatar.jpgPublic-readNo expiry (CDN)
S3 upload flow: Client calls POST /api/upload/audio-presign → Express generates pre-signed PUT URL → Client uploads file directly to S3 → Client sends the returned S3 key back to Express → Express saves the key in MongoDB. Never stream file bytes through your Node.js server.
